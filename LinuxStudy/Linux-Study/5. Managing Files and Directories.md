# **Creating Files**

The <mark>touch</mark> command creates empty files instantly:  
`touch file.txt`  
`touch document.pdf note.md`  
You can create multiple files at once by listing their names separated by spaces.  
The <mark>echo</mark> command can create files with content:  
`echo "Hello World" > greeting.txt`  
`echo "My first file" > text.txt`  


# **Creating Directories**

The <mark>mkdir</mark> command creates new directories in Linux. The name stands for "make directory" and it's one of the most frequently used commands for organising your files.  
To create a single directory, type <mark>mkdir</mark> followed by the directory name:  
`mkdir documents`  
This creates a directory called <mark>documents</mark> in your current location. You can verify it was created using the <mark>ls</mark> command.  
Creating multiple directories at once is possible by listing them after <mark>mkdir</mark>:  
`mkdir photos videos music`  
Sometimes you need to create a directory structure where parent directories don't exist yet. The <mark>-p</mark> option creates parent directories automatically:  
`mkdir -p projects/website/images`  
This command creates the entire path even if <mark>projects</mark> and <mark>website</mark> directories don't exist.  
Directory names should avoid spaces and special characters. Use underscores or hyphens instead:  
`mkdir my_project`  
`mkdir web-development`  
If you must use spaces, wrap the name in quotes:  
`mkdir "my projects"`  
The <mark>mkdir</mark> command will show an error if you try to create a directory that already exists. You can suppress this error with the <mark>-p</mark> option, which makes the command succeed silently if the directory exists.


# **Removing Files and Directories**

The <mark>rm</mark> command removes files and directories from your system. Since Linux doesn't have a recycle bin by default, deleted files are gone permanently, so use this command carefully.  
To remove a single file:  
`rm filename.txt`  
Removing multiple files works by listing them:  
`rm file1.txt file2.txt file3.txt`  
For directories, you need the <mark>-r</mark> option, which stands for "recursive". This removes the directory and everything inside it:  
`rm -r old_documents`  
Empty directories can also be removed with the <mark>rmdir</mark> command:  
`rmdir empty_folder`  
The <mark>rmdir</mark> command only works on empty directories and will show an error if the directory contains files.  
Linux will ask for confirmation when removing  write-protected files. To force removal without prompts, use the <mark>-f</mark> option:  
`rm -f protected.txt`  
Combining <mark>-r</mark> and <mark>-f</mark> creates a powerful but any confirmation prompts.  
Always double-check your command before pressing Enter. A simple typo with <mark>rm -rf</mark> can delete important files. Consider using <mark>ls</mark> first to verify you're targeting the correct files or directories.  


# **Copying Files**

The <mark>cp</mark> command copies files from one location to another:  
`cp source.txt destination.txt`  
`cp file.txt backup.txt`  
Copy files to different directories:  
`cp document.txt Documents/`  
`cp photo.jpg Pictures/vacation/`  
Copy multiple files to a directory:  
`cp dile1.txt file2.txt file3.txt Documents/`  

**Copying  Directories**

To copy entire directories with their contents, use the <mark>-r</mark> flag:  
`cp -r myfolder backupfolder`  
`cp -r Documents/ backup/Documents/`  


# **Renaming and Moving Files**

The <mark>mv</mark> command serves two purpose in Linux:  
moving files to different locations and renaming them. When you use <mark>mv</mark> within the same directory, you're essentially renaming the file.  
To rename a file, use <mark>mv</mark> followed by the current name and the new name:  
`mv oldname.txt newname.txt`  
This changes <mark>oldname.txt</mark> to <mark>newname.txt</mark> in the same directory.  
Renaming directories works the same way:  
`mv old_name new_name`  
You can rename and move a file simultaneously by specifying a different path:  
`mv file.txt documents/document1.txt`  
This move <mark>file.txt</mark> into the <mark>documents</mark> directory and renames it to <mark>document1.txt</mark>.  
Linux treats moving and renaming as the same operation. The <mark>mv</mark> command actually changes the file's path in the filesystem, whether that means a new location, new name, or both.  
If the destination file already exists, <mark>mv</mark> will overwrite it without warning. To get a confirmation prompt before overwriting, use the <mark>-i</mark> option:  
`mv -i old_file.txt already_existing_file.txt`  
The system will ask if you want to overwrite the existing file.  
Unlike copying, moving or renaming with <mark>mv</mark> doesn't create duplicates. The original file disappears and only exists with its new name or location.  


# **Using Wildcards**

Wildcards are special characters that represent patterns in filenames. They help you work with multiple files at once without typing each filename individually.  
The asterisk <mark>*</amrk> matches any number of characters, including none:  
`ls *.txt`  
This list all files ending with <mark>.txt</mark>. The <mark>*</mark> represents any characters before the extension.  
`rm photo*`  
This removes all files starting with "photo", regardless of what follows.  
The question mark <mark>?</mark> matches exactly one character:  
`ls file?.txt`  
This matches <mark>file1.txt</mark>, <mark>file2.txt</mark>, or <mark>fileA.txt</mark>, but not <mark>file10.txt</mark>.  
Square brackets <mark>[]</mark> match any single character from the specified set:  
`ls file[123].txt`  
This matches <mark>file1.txt</mark>, <mark>file2.txt</mark>, or <mark>file3.txt</mark>.  
You can specify ranges inside brackets:  
`ls document[a-z].txt`  
This matches files like <mark>documenta.pdf</mark> through <mark>documentz.pdf</mark>.  
Numbers work the same way:  
`ls report[0-1].doc`  
Wildcards work with most Linux commands:  
`cp *.jps backup/`  
`mv temp* /tmp/`  
`rm draft[1-5].txt`  
The shell expands wildcards before passing filenames to commands. If no files match the pattern, most commands will show an error message.  


# **Understanding File Paths**

File paths tell Linux exactly where to find files and directories in your system. Understanding paths is essential for navigating and managing your files effectively.  
Linux uses forward slashes <mark>/</mark> to separate directories in paths, unlike Windows which uses backslashes.  
An absolute path starts from the root directory <mark>/</mark> and gives the complete location:  
`/home/username/documents/reports/report.txt`  
This path works from anywhere in the system because it starts from the root.  
A relative path starts from your current directory:  
`documents/report.txt`  
If you're in <mark>/home/username</mark>, this relative path points to the same files as the absolute path above.  
The current directory is represented by a single dot <mark>.</mark>:  
`ls .`  
The parent directory uses two dots <mark>..</mark>:  
`cd ..`  
This moves you up one directory level.  
You can combine <mark>..</mark> to move up multiple levels:  
`de ../..`  
The tilde <mark>~</mark> represents your home directory:  
`cd ~/documents`  
This is equivalent to <mark>/home/username/documents</mark>.  
When working with paths containing spaces, wrap then in quotes:  
`cd "my documents"`  
Understanding the difference between absolute and relative paths helps you write commands that work reliably regardless of your current location in the filesystem.  


# **Liking Files**

Links in Linux create connections between files, allowing one file to appear in multiple locations or have multiple names. There are two types of links:  
Hard links create multiple names for the same file. When you create a hard link, both names point to the same data on disk:  
`ln original.txt hardlink.txt`  
Both files are essentially the same file with different names. Deleting one doesn't affect the other because the data remains as long as at least one link exists.  
Symbolic links (also called soft links or symlinks) create a pointer to another file:  
`ln -s original.txt symlink.txt`  
The <mark>-s</mark> option creates a symbolic link. This link points to the original file by name, not by data.  
The key difference:  if you delete the original file, hard link still work because they point to the data itself.  Symbolic links become broken because they point to a filename that no longer exists.  
You can create links to directories, but only symbolic links work for this:  
`ln -s /path/to/directory shortcut`  
Hard links show the same file size and identical content. Symbolic links display an arrow pointing to their target.  
Links are useful for organising files, creating  shortcuts, and maintaining compatibility when moving files to new locations.  


# **Archiving and Compressing Files**

Archiving combines multiple files into a single file, while compression reduces file sizes. Linux provides several tools for these tasks, with <mark>tar</mark> being the most common for archiving.  
The <mark>tar</mark> command creates archives (also called tarballs). To create an archive:  
`tar -ch archive.tar documetns/`  
The <mark>-c</mark> option create an archive, and <mark>-f</mark> specifies the filename. This combines all files in the <mark>documents</mark> directory into <mark>archive.tar</mark>.  
To extract an archive:  
`tar -xf archive.tar`  
The <mark>-x</mark> option extracts files from the archive.  
Adding compression reduces archive size significantly. The <mark>-z</mark> option adds gzip compression:  
`tar -czf archive.tar.gz documents/`  
The <mark>.tar.gz</mark> extension indicates a compressed tar archive.  
To extract compressed archives:  
`tar -xzf archive.tar.gz`  
You can list archive contents without extracting:  
`tar -tzf archive.tar.gz`  
The <mark>-t</mark> option lists files, and  <mark>-z</mark> handles compression.  
For individual file compression, use <mark>gzip</mark>:  
`gzip largefile.txt`  
This creates  <mark>largefile.txt.gz</mark> and removes the original. To decompress:  
`gunzip largefile.txt.gz`  
Archives are perfect for backups, transferring multiple files, or organising project files. Compression saves storage space and reduces transfer times over networks.
