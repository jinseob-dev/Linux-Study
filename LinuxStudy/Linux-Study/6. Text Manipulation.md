# **Extracting specific columns (cut)**

The <mark>cut</mark> command extracts specific parts of each line from text files. Think of it like selecting columns in a spreadsheet. This tool is perfect when you need to grab specific information from structured data.  
The most common usage involves specifying which characters or fields to extract. Use <mark>-c</mark> for character positions and <mark>-f</mark> for fields separated by delimiters.  

**Character-based extraction**

To extract specific character positions:  
`echo "Hello World" | cut -c 1-5`  
Output:  
`Hello`  
You can also extract multiple ranges:  
`echo "Linux Tutorial" | cut -c 1-5,7-14`  
Output:  
`Linux Tutorial`  

**Field-based extraction**

When working with delimited data like CSV files, use <mark>-f</mark> to specify fields and <mark>-d</amrk> to set the delimiter:  
`echo "apple,banana,cherry" | cut -d ',' -f 2`  
Output:  
`banana`  
Extract multiple fields:  
`echo "John:25:Engineer" | cut -d ':' -f 1,3`  
Output:  
`John:Engineer`  

**Working with files**

Create a sample file to practice:  
`echo -e "Name,Age,Job\nAlice,30,Developer\nBob,25,Designer" > people.csv`  
`cut -d ',' -f 1,3 people.csv`  
Output:  
`Name,Job`  
`Alice,Developer`  
`Bob,Designer`  
The <mark>cut</mark> command processes each line independently, making it reliable for extracting consistent data patterns from large files.  


# **Sorting (sort)**

The <mark>sort</mark> command arranges lines of text in a specific order. By default, it sorts alphabetically, but you can customise the sorting behaviour for numbers, dates, and more complex data.  

**Basic sorting**

Sort lines alphabetically:  
`echo -e "banana\napple\ncherry" | sort`  
Output:  
`apple`  
`banana`  
`cherry`  
Reverse the order with <mark>-r</mark>:  
`echo -e "banan\napple\ncherry" | sort -r`  
Output:  
`cherry`  
`banana`  
`apple`  

**Numeric sorting**

When sorting numbers, use <mark>-n</mark> to avoid alphabetical sorting issues:  
`echo -e "10\n2\n100\n1" | sort -n`  
Output:  
`1`  
`2`  
`10`  
`100`  
Without, <mark>-n</mark>, the result would be incorrect:  <mark>1, 10, 100, 2</mark>.  

**Field-based sorting**

Sort by specific columns using <mark>-k</mark> followed by the field number:  
`echo -e "Alice 30\nBob 25\nCharlie 35" | sort -k 2 -n`  
Output:  
`Bob 25`  
`Alice 30`  
`Charlie 35`  
Case-insensitive sorting use <mark>-f</mark>:  
`echo -e "Banana\napple\nCherry" | sort -f`  
Output:  
`apple`  
`Banana`  
`Cherry`  


# **Removing duplicate lines (uniq)**

The <mark>uniq</mark> command removes or reports duplicate lines from text. Important note:  <mark>uniq</mark> only works on consecutive duplicate lines, so you often need to sort your data first.  

**Basic duplicate removal**

Remove consecutive duplicate lines:  
`echo -e "apple\napple\nbanana\nbanana\ncherry" | uniq`  
Output:  
`apple`  
`banana`  
`cherry`  

**Working with unsorted data**

For non-consecutive duplicates, combine with <mark>sort</mark>:  
`echo -e "apple\nbanana\napple\ncherry\nbanana" | sort | uniq`  
Output:  
`apple`  
`banana`  
`cherry`  

**Counting duplicates**

Use <mark>-c</mark> to count how many times each line appears:  
`echo "apple\napple\nbanana\ncherry\nbanana" | sort | uniq -c`  
Output:  
`2 apple`  
`2 banana`  
`1 cherry`  

**Finding only duplicates or unique lines**

Show only lines that appear more than once with <mark>-d</mark>:  
`echo -e "apple\napple\nbanana\ncherry\nbanana" | sort | uniq -d`  
Output:  
`apple`  
`banana`  
Show only lines that appear exactly once with <mark>-u</mark>:  
`echo -e "apple\napple\nbanana\cheery\nbanana" | sort | uniq -u`  
Output:  
`cherry`  

**Ignoring case differences**

Use <mark>-i</mark> to treat uppercase and lowercase as the same:  
`echo "Apple\napple\nBanana\nbanana" | sort | uniq -i`  
Output:  
`Apple`  
`Banana`  


#  **Translating and deleting characters (tr)**

The <mark>tr</mark> command translates or deletes characters from input text. It works character by character, making it perfect for changing case, removing unwanted characters, or replacing specific symbols.  

**Character translation**

Replace one set of characters with another:  
`echo "hello world" | tr 'a-z' 'A-Z'`  
Output:  
`HELLO WORLD`  
Replace specific characters:  
`echo "hello-world-test" | tr '-' '_'`  
Output:  
`hello_world_test`  

**Deleting characters**

Use <mark>-d</mark> to delete specific characters:  
`echo "hello123world456" | tr -d '0-9'`  
Output:  
`helloworld`  
Remove spaces:  
`echo "hello world test" | tr -d ' '`  
Output:  
`helloworldtest`  

**Squeezing characters**

Use <mark>-s</mark> to squeeze multiple consecutive characters into one:  
`echo "hello   world" | tr -s ' '`  
Output:  
`hello world`  

**Character classes**

Use predefined character classes for common operations:  
`echo "Hello World 123!" | tr '[:upper:]' '[:lower:]'`  
Output:  
`hello world 123!`  
Common character classes include <mark>[:alpha:]</mark> (letters), <mark>[:digit:]</mark> (numbers), <mark>[:space:]</mark> (whitespace), and <mark>[:punct:]</mark> (punctuation).  

**Combining operations**

Delete and squeeze in one command:  
`echo "Hello,,,World  123" | tr -d ',' | tr -s ' '`  
Output:  
`Hello World 123`  


# **Counting (wc, nl)

Text counting commands help you analyse file contents and gather statistics. The <mark>wc</mark> command count words, lines, and characters, while <mark>nl</mark> adds line numbers to text.  

**Word count with wc**

The <mark>wc</mark> command shows lines, words, and characters by default:  
`echo -e "Heelo world/nThis is a test/nEnd of text" | wc`  
Output:  
`  3  8  38`  
This shows 3 lines, 8 words, and 38 characters (including newlines).
