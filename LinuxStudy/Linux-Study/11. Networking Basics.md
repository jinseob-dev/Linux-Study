# **Understanding IP Addresses**

IP addresses are unique identifiers that allow devices to communicate over networks. Think of them as postal addresses for computers.  
There are two main versions of IP addresses currently in use. IPv4 addresses look like <mark>192.168.1.100</mark> and consist of four numbers separated by dots. Each number ranges from 0 to 255. IPv6 addresses are longer and use hexadecimal notation, like <mark>2001:0bd8:85a3:0000:0000:8a2e:0370:7334</mark>.  
IP addresses are divided into network and host portions using subnet masks. A subnet mask like <mark>255.255.255.0</mark> (or <mark>/24</mark> in CIDR notation) means the first three octets identify the network, and the last octet identifies individual hosts.  
Private IP address ranges are reserved for local networks:  
- <mark>10.0.0.0</mark> to <mark>10.255.255.255</mark>
- <mark>172.16.0.0.</mark> to <mark>172.31.255.255</mark>
- <mark>192.168.0.0</mark> to <mark>192.168.255.255</mark>
These addresses cannot be routed over the internet directly.  
To find your IP address:  
`ip addr show`  
Look for <mark>inet</mark> entries. You might see multiple addresses: a loopback address (<mark>127.0.0.1</mark>), a private network address, and possibly a public address.  
The loopback address <mark>127.0.0.1</mark> always refers to your own machine and is useful for testing local services.  
Your default gateway, usually your router, connects your local network to the internet through Network Address Translation (NAT), which allows multiple devices to share a single public IP address.  


# **Checking Network Configuration**

Understanding your network setup is essential for troubleshooting connectivity issues and managing your Linux system effectively.  
The <mark>ip</mark> command is the modern tool for viewing network configuration. To see all network interfaces and their status, use:  
`ip addr show`  
This displays information about each network interface, including IP addresses, network masks, and Ethernet states. You'll see entries like <mark>eth0</mark> for Ethernet connections and <mark>wlan0</mark> for wireless interfaces.  
For a simpler view of active interfaces, try:  
`ip link show`  
The older <mark>ifconfig</mark> command still works on many systems:  
`ifconfig`  
To check your routing table and see how network traffic flows, use:  
`ip route show`  
This shows your default gateway and network routes. The default gateway is typically your router's IP address.  
For DNS configuration, examine your the resolver file:  
`cat /etc/resolv.conf`  
This file contains your DNS server addresses, which translate domain names to IP addresses.  
To view network statistics and active connections:  
`ss -tuln`  
The flags mean: <mark>t</mark> for TCP, <mark>u</mark> for UDP, <mark>l</mark> for listening ports, and <mark>n</mark> for numerical addresses instead of resolving hosts.  
These commands provide a complete picture of your network configuration, helping your understand how your system connects to networks and the internet.  


# **Using ping and traceroute**

Network connectivity testing is crucial for diagnosing connection problems. The <mark>ping</mark> and <mark>traceroute</mark> commands are your primary tools for this purpose.  
The <mark>ping</mark> command sends small packets to a destination and measures response time:  
`ping goole.com`  
This sends continuous pings until you stop it with <mark>Ctrl+C</mark>. You'll see output showing response times in milliseconds. Lower times indicate better connectivity.  
To send only a specific number of pings:  
`ping -c 4 google.com`  
The <mark>-c</mark> flag specifies the count. This sends exactly 4 pings then stops.  
To ping with a specific interval:  
`ping -i 2 google.com`  
This pings every 2 seconds instead of the default 1 second.  
The <mark>traceroute</mark> command shows the path packets take to reach a destination:  
`traceroute goole.com`  
You might need to install it first:  
`sudo apt install traceroute # Ubuntu / Debian`  
`sudo dnf install traceroute # RedHat`  
Traceroute displays each router (hop) between your computer and the destination, along with response times. This helps identify where network problems occur.  
An alternative is <mark>tracepath</makr>, which is often pre-installed:  
`tracepath google.com`  
Common ping results and their meanings:  
- Fast responses (under 50ms): Good connection
- Slow response (over 200ms): Network congestion
- Packet loss: Unreliable connection
- "Destination unreachable": Routing problems
- No response: Firewall blocking or destination down
These tools help isolate whether problems are local, with your  ISP, or at the destination.  


# **Configuring Network Interfaces**

Network interface configuration allows you to set IP addresses, enable or disable connections, and manage network settings. This is an essential skill for working with Linux systems, especially servers and minimal installations.  
Modern Linux systems use different methods for network configuration. Many distributions use NetworkManager or systemd-networkd, but understanding manual configuration is valuable.  

**Understanding Interface Names**

Historically, Linux named network interfaces using simple names like <mark>eth0</mark>, <mark>eth1</mark>, <mark>wlan0</mark>, etc. These are still commonly used in tutorials and examples because they are short and easy to recognise.  
However, most modern Linux distributions now use predictable network interface names, which are longer but more consistent across reboots. These names reflect hardware location and may look like:  
- <mark>enp0s3</mark> - Ethernet (PCI bus 0, slot 3)
- <mark>ens33</mark> - Ethernet (slot 33)
- <mark>wlp2s0</mark> - Wireless (PCI bus 2, slot 0)
To find your actual interface name, run:  
`ip link show`  
You'll see output like:  
`2: enp0s3: <`
