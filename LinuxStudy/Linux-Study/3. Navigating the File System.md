# **Understanding the File System Hierarchy**

Linux organise files and directories in a tree-like structure that starts from a single root directory. Unlike Windows with multiple drive letters like C: or D:, Linux uses a unified approach where everything branches out from one main directory called the root, represented by a forward slash <mark>/</mark>.
Think of this structure like an upside-down tree. The root <mark>/</mark>is at the top, and all other directories branch out from it. Every file and directory on your system has a specific location that can be described by its path from the root.
The Linux file system follows standards that make it consistent across different distributions. This means that whether you're using  Ubuntu, Fedora, or any other Linux system, you'll find similar directory structures and organisation.
Each directory serves a specific purpose. For example, user files live in <mark>/home</mark>, system programs are stored in <mark>/bin</mark> and <mark>/usr/bin</mark>, and configuration files are kept in <mark>/etc</mark>. This organised approach makes it easier to locate files and understand how the system works.
When you navigate through directories, you're moving up and down this tree structure. Moving toward the root is going "up" the tree, while moving into subdirectories is going "down". This hierarchical system  keeps everything organised and prevents the chaos that could result from having files scattered randomly across your storage devices.
Understanding this foundation will help you navigate confidently and know where to look for different types of files on any Linux system.

# **Home Directory Explained**

Your home directory is your personal space on a Linux system where you store your files, documents and personal configurations. Every user on the system gets their own home directory, typically located at <mark>/home/username</mark> where username is your actual login name.
When you first log into a Linux system, you automatically start in your home directory. This is your default location and the place where you have full control over files and folders. You can create, modify, and delete files here without needing special permissions.
The home directory contains several important subdirectories that help organise your personal files. Common folders include <mark>Documents</mark>, <mark>Downloads</mark>, <mark>Pictures</mark>, and <mark>Videos</mark>. different default folders, but the concept  remains the same.
Your home directory also stores personal configuration files, most of which are hidden and start with a dot. These files control how your applications behave and store your personal settings. For example, <mark>.bashrc</mark> controls your shell environment, and <mark>.config</mark> contains application-specific settings.
You can always return to your home directory quickly using the  <mark>cd</mark> command without any arguments:  
`cd`  
You can also use the tilde symbol <mark>~</mark> as a shortcut to represent your home directory path:  
`cd ~/Documents`  
This command takes you to the Documents folder inside your home directory, regardless of where you currently are in the file system.


# **Important System Directories**

Linux systems contain several critical directories that serve specific functions. Understanding these directories helps you navigate the system effectively and know where to find different types of files.
The <mark>/dev</mark> directory contains special files that represent devices like disks, terminals, printers, and system resources. When a program reads from or writes to a file in /dev. it's communicating directly with the hardware or kernel drivers.
The <mark>/bin</mark> directory contains essential system programs and commands that all users can execute.
Basic commands like <mark>ls</mark>, <mark>cp</mark>, and <mark>mkdir</mark> live here. These are programs needed for the system to function properly.
The <mark>/etc</mark> directory stores system-wide configuration files. When you need to modify how a service works or change system settings, you'll often find the configuration files here. For example, network settings, user account information, and service configurations are stored in <mark>/etc</mark>.
The <mark>/usr</mark> directory contains user programs and data. Most applications you install end up here, with <mark>/usr/bin</mark> holding user programs and <mark>/usr/share</mark> containing shared data like documentation and application resources.
The <mark>/var</mark> directory holds variable data that changes during system operation. Log files, temporary files, and databases are commonly stored here. When troubleshooting system issues, you'll often check <mark>/var/log</mark> for error messages and system information.
The <mark>/tmp</mark> directory provides temporary storage space for programs and users. Files here are typically deleted when the system restarts, making  it perfect for short-term storage needs.
The <mark>/root</mark> directory serves as the home directory for the root user, which is separate from regular user home directories in <mark>/home</mark>. This separation provides additional security for the system administrator account.


# **Using the File Manager**

Most Linux desktop environments include a graphical file manager that provides an intuitive way to navigate your file system without using command-line tools. The file manager gives you a visual representation of directories and files, similar to Windows Explorer or macOS finder.
Popular file managers include Nautilus in GNOME environments, Dolphin in KDE, and Thunar in XFCE. provide similar core functionality for browsing, copying, moving, and organising files.
The typical file manager window shows a sidebar with quick access to important locations like your home directory, Documents, Downloads, and mounted drives. The main area displays the contents of the current directory, showing both files and folders as icons or in a detailed list view.
You can navigate by double-clicking on folders to enter them, or using the back and forward buttons to move through your browsing history. The address bar shows your current location in the file system and often allows you to type a path directly to jump to a specific location.
Most file managers support tabbed browsing, letting you keep multiple locations open simultaneously. This feature proves helpful when you need to copy or move files between different directories.
Right-clicking on files or folders opens a context menu with common actions like copy, cut, paste, delete, and properties. The properties dialog shows detailed information about files, including size, permissions, and modification dates.
Many file managers also integrate with your desktop environment's search functionality, making it easy to locate files by name or content across your entire system.


# **Hidden Files and Directories**

Linux systems use a simple convention to hide files and directories: any file or folder name that starts with a dot <mark>.</mark> is considered hidden. This approach keeps configuration files and system data out of the way during normal file browsing while still making them accessible when needed.
Hidden files typically contain configuration information for applications or system settings. For example, <mark>.bashrc</mark> controls your shell environment, <mark>.config</mark> store application preferences.
By default, most file managers and the <mark>ls</mark> command don't display hidden files. This design choice reduces clutter and prevents accidental modification of important configuration files by inexperienced users.
