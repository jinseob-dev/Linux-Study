# **Understanding the File System Hierarchy**

Linux organise files and directories in a tree-like structure that starts from a single root directory. Unlike Windows with multiple drive letters like C: or D:, Linux uses a unified approach where everything branches out from one main directory called the root, represented by a forward slash <mark>/</mark>.
Think of this structure like an upside-down tree. The root <mark>/</mark>is at the top, and all other directories branch out from it. Every file and directory on your system has a specific location that can be described by its path from the root.
The Linux file system follows standards that make it consistent across different distributions. This means that whether you're using  Ubuntu, Fedora, or any other Linux system, you'll find similar directory structures and organisation.
Each directory serves a specific purpose. For example, user files live in <mark>/home</mark>, system programs are stored in <mark>/bin</mark> and <mark>/usr/bin</mark>, and configuration files are kept in <mark>/etc</mark>. This organised approach makes it easier to locate files and understand how the system works.
When you navigate through directories, you're moving up and down this tree structure. Moving toward the root is going "up" the tree, while moving into subdirectories is going "down". This hierarchical system  keeps everything organised and prevents the chaos that could result from having files scattered randomly across your storage devices.
Understanding this foundation will help you navigate confidently and know where to look for different types of files on any Linux system.

# **Home Directory Explained**

Your home directory is your personal space on a Linux system where you store your files, documents and personal configurations. Every user on the system gets their own home directory, typically located at <mark>/home/username</mark> where username is your actual login name.
