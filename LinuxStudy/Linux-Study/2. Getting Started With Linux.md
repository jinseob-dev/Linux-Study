# **Choosing a Linux Distribution**

A Linux distribution, or "distro" is a complete operating system built around the Linux kernel. Each distribution packages Linux with different software, desktop environments, and tools to create unique user experiences.
For beginners, three distributions stand out as excellent starting points:

**Ubuntu** remains the most popular choice for newcomers. It offers extensive documentation, a large community for support, and works well on most hardware. Ubuntu uses the GNOME desktop environment by default and receives regular updates every six months.

**Linux Mint** provides a more traditional desktop experience similar to Windows. It's based on Ubuntu but includes multimedia codes and proprietary drivers out of the box. The Cinnamon desktop environment feels familiar to Windows users.

**Fedora** offer cutting-edge features while maintaining stability. It's backed by Red Hat and serves as a testing ground for enterprise technologies. Fedora updates more frequently and includes the latest software versions.

Consider your priorities when choosing. Ubuntu offers the best learning resources and community support. Mint provides the smoothest transition from Windows. Fedora gives you access to the newest features and technologies.

If your goal is to work in enterprise environments or gain experience with Red Hat technologies, you should consider a RHEL-compatible distribution like CentOS Stream, Rocky Linux, or AlmaLinux. These systems behave almost identically to Red Hat Enterprise Linux and are ideal for learning RHEL-specific tools like dnf, SELinux, and systemd. They're widely used in data centres and corporate settings, making them valuable for job preparation.

All three distributions can run on the same hardware and perform similar tasks. Your choice won't lock you into a permanent decision since you can always switch distributions later or run multiple ones on the same computer.

Start with one that appeals to you most. The fundamental Linux skills you lear will transfer between any distribution you choose.


# **Downloading Linux ISOs**

An ISO file is a complete copy of a Linux distribution stored in a single file. This file contains everything needed to install or run Linux on your computer.
Visit the official website of your chosen distribution to download the ISO file. For Ubuntu, go to ubuntu.com and clock the Download section. Linux Mint users should visit linuxmint.com, while Fedora can be found at fedoraproject.org.
Most distributions offer different versions of their ISO files:

**Desktop versions** include a graphical interface and are perfect for everyday use. These typically range from 2-4GB in size.

**Server versions** focus on command-line tools and services without a desktop environment. They're smaller but not recommended for beginners.

**Long Term Support (LTS)** versions receive updates for several years and provide maximum stability.

Choose the 64-bit version unless you have very old hardware. Modern computers from the last 15 years support 64-bit processing and can run  more software.
The download process can take 30 minutes to several hours depending on your internet connection. Ensure you have a stable connection and enough storage space on your computer.
After downloading, verify the file integrity using checksums if provided on the download page. This step confirms your ISo file downloaded correctly and hasn't been corrupted during installation media.

# **Installing Linux on VirtualBox**

VirtualBox is free and open-source virtualisation software that allows you to run Linux inside your current operating system without affecting your main system. It's an excellent way to learn Linux safely and easily.

**Step 1: Download and Install VirtualBox**

First, visit the VirtualBox website and download the version compatible with your operating system, whether it's Windows, macOS, or Linux. Once downloaded, run the installer and follow the instructions to complete the installation. After installation, open VirtualBox to get started.

**Step 2: Download a Linux ISO image**

After you have downloaded the Linux ISO file by following the previous steps, make sure it is saved in a location you can easily find, such as your Downloads folder or a dedicated directory for installation files. This ISO file acts as a virtual DVD containing the full Linux installer.
You will use this ISO image to install Linux inside VirtualBox, so having quick access to it will speed up the setup process. There's no need to create a physical USB or DVD since VirtualBox can boot directly from the ISO file.

**Step 3: Create a New Virtual Machine**

Inside VirtualBox, create a new virtual machine by clicking on the **New** button. Enter a name for your VM, such as "Ubuntu VM", and specify the type as **Linux**. Be sure to select the version that matches your downloaded ISO, for example, Ubuntu 64-bit.

**Step 4: Configure Virtual Machine Settings**

Assign memory (RAM) to your VM; 2048 MB (2GB) is typically enough for basic use with  most Linux distributions. You will also need to create a virtual hard disk. Choose the option to create a virtual hard disk now, then select the Virtual Disk Image (VDI) format. It's best to pick dynamically allocated storage, which saves space on your physical drive. Set the disk size to at least 20 GB for a smooth experience.

**Step 5: Attach the Linux ISO to the VM**

Next, select your VM and open its settings. Navigate to the **Storage** section. Under the **Controller: IDE** area, click the empty CD/DVD icon. Click the disk icon on the right side and select **Choose a disk file**. Then, browse to and select the Linux ISO image you downloaded earlier.

**Step 6: Start the Virtual Machine and Install Linux**

Start your virtual machine. It will boot from the ISO image and load the Linux installer. Follow the Installation process as you would on a physical computer. Select your preferred language and region, then choose the correct keyboard layout. When asked about the installation type, select the option to erase the disk and install Linux-this will only affect the virtual hard disk, so your real computer's files remain safe. Create a user account with a secure password, and let the installer handle disk partitioning automatically. The installation process usually takes between 20 to 40 minutes.

**Step 7: Finalise Installation an Reboot**

Once the installation completes, the installer will prompt you to restart the system. Before rebooting, remove the ISO image from the virtual drive to avoid booting back into the installer. This can be done by selecting **Devices > Optical Drives** in VirtualBox and unchecking the ISO file.  After that, restart the VM. Your Linux system will then boot up, ready to use inside VirtualBox.


# **Creating Bootable USB Drives**

A bootable USB drive allows you to install or test Linux without modifying your computer's hard drive. This process copies the ISO file to a USB drive in a special way that makes your computer recognise it as installation media.
You'll need a USB drive with at least 4GB of storage space. The process will erase everything on the USB drive, so backup any important files first.
**For Windows users**, download Rufus from rufus.ie. This free tool simplifies the process significantly. Insert your USB drive, open Rufus, and select your USB device from the dropdown menu. Click "SELECT" to choose your downloaded ISO file. Leave other settings at their defaults and click "START" to begin  the process.
**For Mac users**, balenaEtcher works excellently and runs on maOS. balena.io/etcher, install the application, then select your ISO file and USB drive. Click "Flash" to create your bootable drive.
**For existing Linux users**, the dd command provides a straightforward solution:
/n`sudo dd if=/path/to/your/linux.iso of=dev/sdX bs=4M status=progress`
Replace <mark>/path/to/your/linux.iso</mark> with your ISO file location and <mark>/dev/sdX</mark> with your USB device identifier.
The creation process typically takes 5-15 minutes. Don't remove the USB drive until the process completes entirely. Your bootable USB drive is now ready for installing or testing Linux.


# **Installing Linux on a Computer**

Installing Linux replaces your current operating system with Linux. Before proceeding, backup all important files since this process will erase your hard drive.
Insert your bootable USB drive and restart your computer. Most computers need you to press a specific key during startup to access the boot menu. Common keys include <mark>F12, F2, ESC, or DEL</mark>. The exact key appears on screen during startup.
Select your USB drive from the boot menu to start the Linux installer. Most distributions provide a graphical installer that guides you through each step.
The installer will ask several important questions:

**Language and location** settings determine your system's default language and timezone.

**Keyboard layout** should match your physical keyboard. Test typing in the provided field to ensure correct layout selection.

**Installation type** offers options for disk usage. Choose "Erase disk and install Linux" for a complete Linux installation. This option removes everything on your hard drive.

**User account creation** requires a username and password. Choose a strong password since this account will have administrative privileges.

**Disk partitioning** happens automatically for beginners, but you can customise it if needed. The installer creates necessary partitions for the system.


# ** Dual Booting with Windows**

Dual booting lets your keep Windows and Linux on the same computer, choosing which operating system to use each time you start up. This approach provides environment while exploring Linux.
Before starting, backup your important Windows files and create a Windows recovery drive. Shrink your Windows partition to make space for Linux using Windows Disk management tool. Right-click "This PC", select "Manage", then "Disk Management". Right-click  your main drive and choose "Shrink Volume". Allocate at least 50 GB for a comfortable Linux installation.
Boot from your Linux USB drive and start the installation process. When you reach the installation type screen, select "Install alongside Windows" or "Something else" for manual partitioning.
The automatic option detects Windows and creates appropriate partitions for Linux. It happens the technical details and sets up the boot loader to present both operating systems at startup.
For manual partitioning, creates these partitions in your free space:
A root partition (<mark>/</mark>) with at least 20 GB using ext4 filesystem A swap partition equal to your RAM size Optionally, a separate home partition (<mark>/home</mark>) for personal files
The installer configures GRUB,  a boot loader that presents a menu at startup. This menu lets you choose between Windows and Linux each time you boot your computer.
After installation, you'll see the GRUB menu for a few seconds during startup, defaulting to Linux. Use arrow keys to select Windows when needed.


# **First Boot and Initial Setup**

Your first Linux boot presents a clean desktop environment ready for customisation and exploration. The initial setup precess varies slightly between distributions but covers similar essential areas.
Most distributions launch a welcome application or setup wizard automatically. This tool guides you through important first-time configurations and introduces key features of your new system.

**Software updates** should be your first priority. Linux distributions regularly release security patches and improvements. Look for an update manager application or notification indicating available updates. Install all recommended updates and restart if prompted.

**Additional drivers** might be needed for optimal hardware performance. Ubuntu and similar distributions include a "Additional Drivers" tool that detects and installs proprietary drivers for graphics cards, WIFI adapters, and other hardware components.

**Software installation** introduces you to your distribution's package manager. Ubuntu uses the Software Centre, while other distributions have their own app stores. Browse available applications and install essential software like web browsers, media players, or office suites.

**System settings** let you personalise your desktop experience. Adjust display resolution, configure WIFI networks, set up user accounts and customise appearance themes. Most settings are accessible through a centralised settings application.
