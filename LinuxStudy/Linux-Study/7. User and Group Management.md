
# **Understanding Users and Groups**

Linux is a multi-user operating system, meaning multiple people can use the same computer simultaneously. To organise and control access, Linux uses users and groups.  

**What is a User?**

Every person who uses a Linux system has a user account. Each user has:  
- A unique username (like "alice" or "sarah")
- A user ID number (UID)
- A home directory (usually <mark>/home/username</mark>)
- A default shell program
The system creates some special users automatically. The most important thing is <mark>root</mark>, which has complete control over the system.  

**What is a Group?**

Groups are collections of users. They make it easier to manage permissions for multiple people at once. Instead of giving permissions to each user individually, you can assign permissions to a group.  
Every user belongs to at least one group called their "primary group". Users can also belong to additional "secondary groups".  

**Viewing User information**

To see information about your current system:  
`whoami`  
`id`  
To see all users on the system:  
`cat /etc/passwd`  
To see all groups:  
`cayt /etc/group`  
The <mark>/etc/passwd</mark> file contains user account information, while <mark>/etc/group</mark> contains group information. These files are readable by everyone but can only be modified by the system administrator.  
Understanding users and groups is fundamental to Linux security and file management.


# **Adding and Removing Users**

Managing user accounts is a common administrative task. You need administrator pricileges (using <mark>sudo</mark>) to add or remove users.  

**Adding a New User**

The <mark>useradd</mark> command creates new user accounts:  
`sudo useradd username`  
This creates a basic account, but it's better to use additional options:  
`sudo useradd -m -s /bin/bash username`  
The <mark>-m</mark> option creates a home directory, and <mark>-s /bin/bash</mark> sets the default shell.  

**Setting a Password**

New users need passwords before they can log in:  
`sudo passwd username`  
The system will prompt you enter and confirm the new password.  

**Adding Users with More Options**

You can specify additional information when creating users:  
`sudo useradd -m -s /bin/bash -c "Full Name" -G groupname username`  
The <mark>-c</mark> option adds a comment (usually the person's full name), and <mark>-G</mark> adds them to additional groups.  

**Removing Users**

To remove a user account:  
`sudo userdel username` 
To remove the user and their home directory:  
`sudo userdel -r username`  
The <mark>-r</mark> option removes the user's home directory and mail files. Be careful with this option as it permanently deletes the user's files.  
Always verify the username before removing accounts to avoid accidentally deleting the wrong user.


# **Modifying User Accounts**

After creating user accounts, you might need to modify their properties. The <mark>usermod</mark> command handles most user account modifications.  

**Changing User Information**

To change a user's full name or comment:  
`sudo usermod -c "New Full Name" username`  
To change a user's default shell:  
`sudo usermod -s /bin/zsh username`  

**Modifying Home Directory**

To change a user's home directory:  
`sudo usermod -d /new/home/path username`  
To move the contents to the new location as well:  
`sudo usermod -d /new/home/path -m username`  

**Managing User Groups**

To change a user's primary group:  
`sudo usermod -g newgroup username`  
To add a user to additional groups:  
`sudo usermod -a -G group1,group2 username`  
The <mark>-a</mark> option appends to existing groups. Without it , the user would be removed from all other secondary groups.  

**Locking and Unlocking Accounts**

To temporarily disable a user account:  
`sudo usermod -L username`  
To unlock the account:  
`sudo usermod -U username`  

**Changing Passwords**

Users can change their own passwords:  
`passwd`  
Administrators can change any user's password:  
`sudo passwd username`  
These modifications take effect immediately, though users might need to log out and back in to see some changes.
