# **Understanding Permission Notation**

Linux uses a three-part permission system to control file and directory access. Each file has permissions for three types of users: the owner, the group, and everyone else (others).  
When you run <mark>ls -l</mark>, you see permissions displayed as a string of 10 characters. The first character indicates the file type (d for directory, - for regular file). The remaining 9 characters represent permissions in groups of three.  
`ls -l file.txt`  
`-rw-r--r-- 1 alice users. 1234 Nov 15 10:30 file.txt`  
The permission string <mark>-rw-r--r--</mark> breaks down as:  
- First three characters (rw-): owner permissions  
- Next three characters (r--): group permissions  
- Last three characters (r--): other permissions  
Each group of three represents read ®, write (w), and execute (x) permissions in that order. A dash (-) means the permission is not granted.  

**Numeric Notation**

Permissions can also be expressed as numbers:  
- Read = 4  
- Write = 2  
- Execute = 1  
Add these values together for each user type. For example, <mark>rw-r--r--</mark> becomes 644:  
- Owner: read(4) + write(2) = 6  
- Group: read(4) = 4  
- Other: read(4) = 4  
Common permission combinations include 755 for executable files and 644 for regular files.  


# **Changing Permissions with chmod**

The <mark>chmod</mark> command modifies file and directory permissions. You can see either symbolic or numeric notation to specify the new permissions.  

**Symbolic Notation**
Symbolic notation uses letters to represent users and permissions:  
- u = user (owner)  
- g = group  
- o = others  
- a = all (user, group, others)  
Operations include:  
- + adds permissions  
- - removes permissions  
- = sets exact permissions  
`chmod u+x script.sh # Add execute for owner`  
`chmod g-w document.txt # Remove write for group`  
`chmod o+r readme.txt # Add read for others`  
`chmod a+r public.txt # Add read for everyone`  
You can combine multiple changes in one command:  
`chmod u+rw,g+r,o-rwx private.txt`  

**Numeric Notation**

Using numeric notation, specify the three-digit permission code:  
`chmod 755 script.sh # rwxr-xr-x`  
`chmod 644 document.txt # rw-r--r--`  
`chmod 600 secret.txt # rw-------`  
`chmod 777 shared_folder # rwxrwxrwx`  

**Recursive Changes**

Use the <mark>-R</mark> flag to apply permissions recursively to directories and their contents:  
`chmod -R 755 /home/user/public/`  


# **Changing Ownership with chown**

The <mark>chown</mark> command changes file and directory ownership. Only the root user or the current owner can change ownership, and typically only root can change ownership to another user.  

**Basic Usage**
Change the owner of a file:  
`chown alice document.txt`  
Change both owner and group using a colon:  
`chown alice:students project.txt`  
Change only the group by omitting the username:  
`chown :staff document.txt`  

**Practical Examples**

Transfer ownership of a user's home directory:  
`sudo chown -R newuser:newgtoup /home/newuser/`  
Change ownership of web server files:  
`sudo chown www-dataLwww-data /var/www/html/index.html`  

**Recursive Ownership Changes**

Use <mark>-R</mark> to change ownership recursively for directories:  
`sudo chown -R alice:projects /home/shared/project_files/`  

**Checking Current Ownership**

View current ownership with <mark>ls -l</mark>:  
`ls -l file.txt`  
`-rw-r--r-- 1 alice students 2048 Nov 15 14:22 file.txt`  
The output shows alice as the owner and students as the group.  

**Reference Method**

Copy ownership from another file using the <mark>--reference</mark> option:  
`chown --reference=template.txt newfile.txt`  
This sets the ownership of <mark>newfile.txt</mark> to match <mark>template.txt</mark>.
