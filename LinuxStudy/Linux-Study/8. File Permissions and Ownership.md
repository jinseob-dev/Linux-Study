# **Understanding Permission Notation**

Linux uses a three-part permission system to control file and directory access. Each file has permissions for three types of users: the owner, the group, and everyone else (others).  
When you run <mark>ls -l</mark>, you see permissions displayed as a string of 10 characters. The first character indicates the file type (d for directory, - for regular file). The remaining 9 characters represent permissions in groups of three.  
`ls -l file.txt`  
`-rw-r--r-- 1 alice users. 1234 Nov 15 10:30 file.txt`  
The permission string <mark>-rw-r--r--</mark> breaks down as:  
- First three characters (rw-): owner permissions  
- Next three characters (r--): group permissions  
- Last three characters (r--): other permissions  
Each group of three represents read ®, write (w), and execute (x) permissions in that order. A dash (-) means the permission is not granted.  

**Numeric Notation**

Permissions can also be expressed as numbers:  
- Read = 4  
- Write = 2  
- Execute = 1  
Add these values together for each user type. For example, <mark>rw-r--r--</mark> becomes 644:  
- Owner: read(4) + write(2) = 6  
- Group: read(4) = 4  
- Other: read(4) = 4  
Common permission combinations include 755 for executable files and 644 for regular files.
