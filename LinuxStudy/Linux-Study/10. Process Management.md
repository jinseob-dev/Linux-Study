**Understanding Processes**
A process is a running program on your Linux system. Every time you execute a command, open an application, or run a script, Linux creates a process for it.  
Each process gets a unique identifier called a Process ID or <mark>PID</mark>. Think of it like a name tag that helps the system keep track of what's running.  
Processes have a parent-child relationship. When you open a terminal, it creates a shell process. Any command you run from that terminal becomes a child process of the shell.  

**Process States**

Processes can be in different states:  
- **Running**: Currently using the CPU
- **Sleeping**: Waiting for something (like user input)
- **Stopped**: Paused by a signal
- **Zombie**: Finished but still in the process table
Every process has an owner-the user who started it. This determines what files and recourses the process can access.  
Processes also have priority levels that determine how much CPU time they get. Higher priority processes get more attention from the system.  
You can see basic process information about your current shell by typing:  
`echo $$ # Shows yout current shell's PID`  
`ps # Shows processes in your current terminal`  
Understanding processes is essential because it helps you monitor system performance, troubleshoot problems, and manage running applications effectively.  


# **Viewing Processes with ps and top**

The <mark>ps</mark> command shows a snapshot of currently running processes. By default, it only shows processes in your current terminal session.  
`ps`  
To see all processes running on your system, use:  
`ps aux`  
This shows processes for all user (<mark>a</mark>), includes processes without terminals (<mark>x</mark>), and displays user information (<mark>u</mark>).  
The output columns include:  
- <mark>USER</mark>: Who owns the process
- <mark>PID</mark>: Process ID
- <mark>%CPU</mark>: CPU usage percentage
- <mark>%MEM</mark>: Memory usage percentage
- <mark>COMMAND</mark>: The command that started the process

**Using top Command**

The <mark>top</mark> command provides a live, updating view of processes:  
`top`  
Unlike <mark>ps</mark>, <mark>top</mark> refreshes automatically every few seconds. It shows the most CPU-intensive processes at the top.  
Key information in <mark>top</mark>:
- Load average and CPU usage
- Memory and swap usage
- Real-time process activity

**Useful Navigation in top**

While <mark>top</mark> is running:  
- Press <mark>q</mark> to quit
- Press <mark>k</mark> to kill a process
- Press <mark>M</mark> to sort by memory usage
- Press <mark>P</mark> to sort by CPU usage
For a simpler alternative, try <mark>htop</mark> if available:  
`htop`  
The <mark>htop</mark> command offers a more colourful and user-friendly interface with mouse support.


# **Killing Processes**

Sometimes processes become unresponsive or consume too many resources. Linux provides several ways to terminate processes.  
The <mark>kill</mark> command sends signals to processes. The most common signal is <mark>SIGTERM</mark>, which politely asks a process to terminate:  
`kill PID`  
Replace <mark>PID</mark> with the actual process ID. For example:  
`kill 1234`  

**Force Killing Processes**

If a process doesn't respond to the polite request, use the <mark>SIGKILL</mark> signal to force termination:  
`kill -9 PID`  
The <mark>-9</mark> option sends <mark>SIGKILL</mark>, which cannot be ignored by the process.  

**Killing by Process Name**

The <mark>killall</mark> command terminates processes by name:  
`killall firefox`  
This kills all processes named <mark>firefox</mark>.  
For more precise control, use <mark>pkill</mark>:  
`pkill -f "python script.py"`  
This kills processes whose command line matches the pattern.  

**Finding Process IDs**

Use <mark>pgrep</mark> to find PIDs by process name:  
`pgrep firefox`  
You can combine commands to kill processes:  
`kill $(pgrep firefox)`  
Always be careful when killing processes, especially system processes, as this can make your system unstable.  


# **Foreground and Background Processes**

When you run a command in the terminal, it normally runs in the foreground, meaning it occupies your terminal until it completes.  
`sleep 30 # This blocks your terminal for 30 seconds`  

**Running Processes in Background**

Add an ampersand <mark>&</mark> at the end of a command to run it in the background:  
`sleep 30 &`  
This immediately returns control to your terminal while the process runs in the background.  

**Managing Background Jobs**

Use the <mark>jobs</mark> command to see background processes started from your current terminal:  
`jobs`  
Each job gets a job number like <mark>[1]</mark> or <mark>[2]</mark>.  

**Bringing Jobs to Foreground**

Use <mark>fg</mark> to bring a background job to the foreground:  
`fg %1 # Brings job 1 to foreground`  
If you have only one background job, simply use:  
`fg`  

**Suspending and Resuming Processes**

Press <mark>Ctrl+Z</mark> to suspend a running foreground process. This pauses it and gives you back the terminal.  
Resume a suspended job in the background with:  
`bg %1 # Resume job 1 in background`  
Resume in the foreground with:  
`fg %1 # Resume job 1 in foreground`  
This job control system lets you multitask effectively within a single terminal session.  


# **Using nice and renice**

Process priority in Linux ranges from -20 (highest priority) to 19 (lowest priority). The default priority is 0. Lower numbers mean higher priority and more CPU time.  
The <mark>nice</mark> command starts a process with a specific priority:  
`nice -n 19 command`  
This runs <mark>command</mark> with a priority of 10 (lower than default). Only root can set negative values (higher priority).  

**Examples of Using nice**

Start a CPU-intensive task with low priority:  
`nice -n 15 find / -name "*.txt" > results.txt &`  
This prevents the search from slowing down other processes.  

**Changing Priority of Running Processes**

Use <mark>renice</mark> to change the priority of an already running process:  
`renice 10 -p PID`  
This sets the process with <mark>PID</mark> to priority 10.  
You can also change priority by process name:  
`renice 5 -u username`  
This changes priority for all processes owned for <mark>username</mark>.  

**Checking Process Priority**

View current priorities using <mark>ps</makr>:  
`ps -eo pid,ni,comm`  
The <mark>NI</mark> column shows the nice value (priority).  
In <mark>top</mark>, the <mark>NI</mark> column also displays priority values.  

**Practical Usage**

Use higher nice values (lower priority) for:  
- File backups
- Large file operations
- Non-urgent batch processing
This ensures interactive applications remain responsive while background tasks run.  


# **Monitoring System Resources**

Linux provides several tools to monitor CPU, memory, disk, and network usage. Understanding resource consumption helps identify performance bottlenecks.  

**CPU and Memory Overview**

The <mark>free</mark> command shows memory usage:  
`free -h # Human-readable format`  
This displays total, used, and available memory plus swap space.  
For CPU information, use:  
`lscpu # Shows CPU details`  

**Real-time Monitoring**

We covered <mark>top</mark> earlier, buy here are additional options for system monitoring:  
`vmstat 2 # Updates every 2 seocnds`  
The <mark>vmstat</mark> command shows virtual memory statistics, CPU usage, and system activity.  

**Disk Usage Monitoring**

Check disk space with:  
`df -h # Shows filesystem usage`  
Monitor disk I/O activity:  
`iostat 2 # Updates every 2 seconds`  

**Process Resource Usage**

See which processes use the most resources:  
`ps aux --sort=-%cpu | head -10 # Top CPU users`  
`ps aux --sort=-%mem | head -10 # Top memory users`  

**Continuous Monitoring**

For ongoing monitoring, combine tools:  
`watch -n 2 'free -h && df -f'`  
This updates memory and disk information every 2 seconds.  
The <mark>watch</mark> command repeats any command at specified intervals, making it useful for monitoring changing  values.  


# **Automating Tasks with cron**

The <mark>cron</mark> service runs commands automatically at scheduled times. This is perfect for backups, maintenance tasks, and regular system operations.  

**Understanding Crontab**

Each user has a personal crontab (cron table) that defines their scheduled tasks. Edit your crontab with:  
`crontab -e`  
View your current crontab:  
`crontab -l`  

**Cron Time Format**

Cron uses five time fields:  
`minute hour day month weekday command`  
- Minute: 0-59
- Hour: 0-23 (24-hour format)
- Day:1-31
- Month:1-12
- Weekday:0-7 (0 and 7 are Sunday)

**Common Cron Examples**

Run a backup every dat at 2:30 AM:  
`30 2 * * * /home/user/backup.sh`  
Run a script every Monday at 9:00 AM:  
`0 9 * * 1 /home/user/weekly_report.sh`  
Run a command every 15 minutes:  
`*/15 * * * * /use/bin/sheck_system.sh`
